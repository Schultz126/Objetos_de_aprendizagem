Como funciona a busca binária?

Ideia geral

A busca binária é um algoritmo eficiente para localizar um elemento em uma coleção ordenada (normalmente em um vetor ou lista ordenada).

A ideia central é dividir o espaço de busca pela metade a cada passo, descartando sempre uma das metades, até que o elemento seja encontrado ou se conclua que não está presente.

Isso contrasta com a busca linear, que examina os elementos um a um, levando tempo proporcional ao tamanho da coleção.

Funcionamento passo a passo

1. Definir os limites de busca

Suponha que temos um vetor ordenado em ordem crescente com n elementos:

  • Definir limites de busca:
  • início = 0 (primeiro índice)
  • fim = n - 1 (último índice)

2. Calcular elemento do meio

  • meio = (início + fim) / 2

3. Comparar o valor procurado (x) com o elemento do meio (A[meio]):

  • Se A[meio] == x, o elemento foi encontrado.
  • Se A[meio] > x, significa que o valor está à esquerda, então ajustamos fim = meio - 1.
  • Se A[meio] < x, significa que o valor está à direita, então ajustamos início = meio + 1.

4.Repetir o processo enquanto início <= fim.

  • Se os limites se cruzarem sem encontrar o elemento, significa que o valor não está presente.

Complexidade

Complexidade de tempo

A cada iteração, o algoritmo descarta metade dos elementos.

  • Após 1 passo → restam n/2 elementos
  • Após 2 passos → restam n/4 elementos
  • Após k passos → restam n / 2^k elementos

O processo termina quando resta somente um elemento: n / 2^k = 1 → 2^k = n → k = log₂(n)

Portanto:

  • Melhor caso: O elemento está exatamente no meio → O(1)
  • Caso médio e pior caso: Precisa reduzir até 1 elemento → O(log n)

Complexidade de espaço

  • Implementação iterativa → usa apenas variáveis auxiliares → O(1)
  • Implementação recursiva → consome memória de pilha proporcional à profundidade da recursão → O(log n)

Condições de uso

  • O vetor precisa estar ordenado.
  • O acesso aos elementos deve ser eficiente (ex.: vetores permitem acesso direto O(1)).
