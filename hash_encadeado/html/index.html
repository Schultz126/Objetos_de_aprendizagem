O que é hash?

De modo simplificado, hash é uma função matemática que recebe uma entrada (texto, número, arquivo) e a transforma em uma saída de tamanho fixo. Essa saída é frequentemente uma sequência curta de bits, conhecida como código hash.

Objetivo de se usar hash

Um dos maiores desafios ao trabalhar com um grande número de dados é a forma de como obter acesso a eles. O uso de tabelas hash é uma solução quando se objetiva alta performance na inserção, busca e remoção de dados, aproximando-se de um tempo de acesso constante (O(1)).

Como funciona e Propriedades Chave

A função hash utiliza operações matemáticas para transformar uma entrada de dados de maneira aparentemente aleatória, mapeando-a para um índice dentro de uma tabela de tamanho fixo.

  • Saídas de mesmo tamanho: O código hash gerado tem sempre o mesmo comprimento, independentemente do tamanho da entrada original.
  • Mesma entrada gera sempre a mesma saída: A função hash é determinística; inserir os mesmos dados duas vezes resultará exatamente no mesmo código hash e, consequentemente, no mesmo índice de tabela.
  • Resistência à Colisão (Ideal): É extremamente difícil, ou idealmente impossível, encontrar duas entradas diferentes que gerem a mesma saída (o mesmo código hash). No entanto, em tabelas de tamanho limitado, colisões (duas chaves mapeando para o mesmo índice) são inevitáveis.

Para minimizar o problema das colisões, é comum desenvolvedores optarem por números primos ao escreverem suas funções hash. Tais números garantem padrões menos repetitivos, como por exemplo, potências de 2. O exemplo prático presente neste site utiliza o número 7 para sua função hash, e ainda que seja em pequena escala, auxilia na compreensão do hash

Aplicações Práticas

Funções hash são vitais em diversas áreas da computação, incluindo:

  • Segurança de Dados: Armazenamento seguro de senhas (nunca se armazena a senha original, mas sim o seu hash).
  • Verificação de Integridade: Checagem rápida de arquivos; se um arquivo for alterado, seu hash muda, indicando corrupção.
  • Estruturas de Dados: Implementação eficiente de dicionários ou mapas (key-value stores).

Tabela Hash Encadeada (Chained Hashing)

Quando duas chaves diferentes (entradas) resultam no mesmo índice na tabela (uma colisão), a Tabela Hash Encadeada resolve o problema usando listas ligadas.

Em vez de armazenar o item diretamente no índice, cada posição da tabela (cada "bucket") armazena o cabeçalho de uma lista ligada. Se ocorrer uma colisão, o novo item é simplesmente adicionado ao final dessa lista ligada, como demonstrado na visualização acima.
